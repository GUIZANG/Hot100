# 模型 1 · 哈希查找、计数与分组

## 一句话识别

题目需要反复询问以下问题时，优先考虑哈希表：

- 某个值以前是否出现过？
- 某个值出现了几次？
- 某个值对应哪个下标？
- 哪些元素属于同一组？

哈希表使用额外空间，把平均查找时间从 `O(n)` 降到 `O(1)`。

## C++ 最小语法

```cpp
#include <unordered_map>

unordered_map<int, int> table;

table[10] = 3;          // 保存：键 10 对应值 3
int value = table[10];  // 读取键 10 对应的值

if (table.count(10)) {
    // 键 10 存在
}
```

在“两数之和”中：

- 键：已经见过的数字
- 值：该数字对应的数组下标

注意：`table[key]` 在键不存在时会自动创建该键。只检查是否存在时，先用 `count`。

## 练习 1A：两数之和

实现 LeetCode 1：

```cpp
vector<int> twoSum(vector<int>& nums, int target) 
```

示例：

```text
输入：nums = [2, 7, 11, 15], target = 9
输出：[0, 1]
```

条件：

- 每组输入有且只有一个答案。
- 同一个位置不能使用两次。
- 需要返回两个元素的下标。

### 写代码前先确定

遍历到 `nums[i]` 时：

1. 它还缺少的数字是 `target - nums[i]`。
2. 在哈希表中查询这个数字是否已经出现。
3. 如果出现，返回“以前的下标”和当前下标。
4. 如果没有出现，再把当前数字和下标存入哈希表。

为什么必须“先查再存”？请在提交代码时一起回答。

### 要求

1. 时间复杂度达到平均 `O(n)`。
2. 不使用两层循环。
3. 在 `01A-两数之和.cpp` 中完成。
4. 完成后保存并提交“练习 1A 写完了”。

### 练习结果

提示后通过。

```text

value = 下标
```

## 练习 1C：字母异位词分组（LeetCode 49）

字母异位词由相同字母重新排列得到，例如：

```text
"eat"、"tea"、"ate"
```

实现：

```cpp
vector<vector<string>> groupAnagrams(vector<string>& strs)
```

示例：

```text
输入：["eat","tea","tan","ate","nat","bat"]
输出：[["bat"],["nat","tan"],["ate","eat","tea"]]
```

分组顺序不作要求。

### 模型识别

同一组单词排序后会得到相同字符串：

```text
"eat" → "aet"
"tea" → "aet"
"ate" → "aet"
```

固定映射关系：

```text
key   = 排序后的字符串
value = 属于这一组的原字符串数组
```

### 新 C++ 语法

```cpp
#include <algorithm>
#include <string>

string key = s;
sort(key.begin(), key.end());

unordered_map<string, vector<string>> map;
map[key].push_back(s);
```

遍历哈希表时，每个元素包含：

```cpp
pair.first   // key
pair.second  // value
```

### 要求

1. 不修改输入中的原字符串。
2. 对每个字符串生成排序后的 `key`。
3. 使用哈希表完成分组。
4. 最后把哈希表中的每个 `value` 加入二维结果数组。
5. 在 `01C-字母异位词分组.cpp` 中完成。
6. 保存后提交“练习 1C 写完了”。

### 练习结果

修正键值对读取方式后通过。

固定映射关系：

```text
key   = 排序后的字符串
value = 同组原字符串数组
```

## 练习 1D：最长连续序列（LeetCode 128）

实现：

```cpp
int longestConsecutive(vector<int>& nums)
```

找出数字连续的最长序列长度，要求平均 `O(n)`。

示例：

```text
输入：[100, 4, 200, 1, 3, 2]
输出：4
解释：最长连续序列为 [1, 2, 3, 4]
```

### 新工具：unordered_set

只关心“某个数字是否存在”，不需要为它保存额外信息时使用：

```cpp
#include <unordered_set>

unordered_set<int> set(nums.begin(), nums.end());

if (set.count(x)) {
    // x 存在
}
```

`unordered_set` 只有 key，没有 value。

### O(n) 的关键

不能从每个数字都向后查找。只有当 `num - 1` 不存在时，`num` 才是一个连续序列的起点：

```text
集合中有 1、2、3、4

1：0 不存在，是起点，从这里查
2：1 存在，不是起点，跳过
3：2 存在，不是起点，跳过
4：3 存在，不是起点，跳过
```

### 要求

1. 先用 `unordered_set` 去重。
2. 只从序列起点向后统计。
3. 返回最长长度。
4. 不允许排序。
5. 在 `01D-最长连续序列.cpp` 中完成。
6. 保存后提交“练习 1D 写完了”。

### 练习结果

提示后通过。已理解：

- `unordered_set` 只有 key，没有 value。
- 集合不能使用下标访问，需要范围循环。
- 只从 `num - 1` 不存在的起点向后查找。
